## What shipped, grouped by surface

### Episodes - Music Videos removed (commit `8df07a6`)

Music videos already have a dedicated tab under `/onchain`, so surfacing them again on `/episodes` was redundant and cluttered the CoverFlow. The `/episodes` tab now shows Sessions + extras (trailers, ending credits, genesis) only. Removed the "Music Videos" category chip, narrowed the `showCategory` state type, and dropped the now-unused icon imports.

### FRIDAY PRESS - Brave Shields fix attempted, then reverted (commits `da12686`, `7d9cbf8`)

Brave Shields blocks the `dweb.link` subdomain CID iframe pattern by default, so the PDF viewer rendered "This page has been blocked by Brave" for some users. Switched the iframe to the `inbrowser.link` service-worker gateway (commit `da12686`), then reverted per request (commit `7d9cbf8`) - the viewer is back on dweb.link direct PDF rendering. If Brave blocking resurfaces, the options are: inbrowser.link, self-hosting the PDFs, or linking out to the PDF directly.

### Music Videos - scrub/fullscreen fix (commit `4297b51`)

The `<video>` element wired its `onPause` handler to the parent's "remove active card" callback, which unmounts the player. But the HTML `pause` event also fires during scrubbing/seeking and on some fullscreen transitions - so pressing fullscreen and skipping ahead kicked the user out mid-skip. Fix: dropped the `onPause` -> parent wiring. The header pause/close button still closes explicitly, and the "one card at a time" rule still works via imperative `videoRef.current.pause()`. `onEnded` -> close stays.

### Onchain - full @thepark Zora catalog imported (commit `3ac9b48`)

**The long-blocked crawl, solved.** Zora's profile collection loads via an authed GraphQL call, but the Coins SDK API at `api-sdk.zora.engineering` serves the same data **unauthenticated**. Pulled `@thepark`'s complete created-coins feed (`profileCoins` query, paginated 20/page): **129 unique Base content coins**, newest first.

- 129 new `Track` entries tagged `series:'zora-coin'` (80 video, 49 image/text)
- Covers + video resolve through dweb.link (CIDs decoded from Zora's CDN preview URLs); every `mintUrl` keeps the house referrer and points at `zora.co/coin/base:0x...`
- `getMockTracksByPlatform('onchain')` folds the coin feed into the All Onchain grid and excludes the FRIDAY PRESS + Music Videos series (they keep their tabs)
- `series` union gains `'zora-coin'`; TrackCard routes coin cards to `/track/[id]` with a play affordance; TrackDetail renders an inline video for video coins

### Onchain - OpenSea collection songs added (commit `7f15e19`)

Two more onchain music sources crawled and added to the All Onchain grid, all inline-playable.

- **the-park-music** (Sound.xyz on Ethereum, `0x21f5...dc3b9`): it's 3 songs sold as editions - "Do It" was already cataloged, so **Rainwater** and **PB&J** (feat. G Flip) were added. Audio + artwork resolved from the on-chain Arweave metadata bundle, served from arweave.net with full band credits.
- **fridays-at-the-park-season-001** (ERC-1155 on Zora, `0x91dc...25b4`): enumerated via on-chain `uri(tokenId)` -> IPFS. **16 audio songs** (DO IT, You're So High, WATER, Onchain Summer, Imagine, Rooms in the House 1-4, ...), streamed via dweb.link with byte-range support. The trailer / artifact / ETH-drop tokens (video/image) were skipped - the ask was songs.

How they were found: OpenSea's collection pages embed a Relay store (`collectionItems.items[]`) with resolved names + media; combined with on-chain `tokenURI`/`uri()` reads + Arweave/IPFS metadata for permanent audio sources.

### Copy - em dashes eliminated (commit `c039d54`)

Replaced every em dash with a standard hyphen across all source files (**144 occurrences in 33 files**) - rendered copy (track descriptions, titles, headings, empty states), accessibility attributes (`aria-label`, `alt`, `title`), and code comments. Verified zero em dashes remain in src and on the live pages.

### Video - Cloudflare Stream HLS for fast playback (commit `71899f5`)

The 80 Zora content-coin videos were playing from their raw IPFS source - large QuickTime `.mov` files off a public gateway, slow to first frame on a cold cache. Zora's API also exposes a **Cloudflare Stream HLS manifest** per coin, which starts almost instantly and adapts to the connection.

- `Track` type gains `hlsUrl`; injected the Cloudflare Stream `.m3u8` manifest into all 80 video-coin entries (raw IPFS kept as fallback)
- New `<HlsVideo>` component: native HLS on Safari/iOS, lazy-loads `hls.js` for MSE browsers (Chrome/Firefox) **only when a video actually plays**, falls back to the progressive IPFS file on error
- TrackDetail inline player uses `<HlsVideo>`; CSP `media-src`/`connect-src` allow `https://*.cloudflarestream.com` (+ `blob:` for MSE)
- Added the `hls.js` dependency (lockfile updated so Vercel `npm ci` stays in sync)
- The 5 older music-video drops (other contracts, no Cloudflare manifest) keep their mp4 source

## How the crawls were cracked (reference)

- **Zora profile:** `https://api-sdk.zora.engineering/profileCoins?identifier=thepark&count=50&after=<cursor>` - unauthenticated, paginated, returns name/address/description/`tokenUri`/`mediaContent` (incl. `videoHlsUrl`).
- **OpenSea collections:** collection page HTML embeds a Relay store at `collectionItems.items[]` with resolved `name`/`animationUrl`/`imageUrl`/`tokenUri`; contract addresses cross-checked on-chain.
- **On-chain metadata:** `tokenURI(uint256)` (`0xc87b56dd`) for ERC-721 / Sound.xyz, `uri(uint256)` (`0x0e89341c`) for ERC-1155; `ar://` resolved via permagate.io (manifest paths) / arweave.net (single txids), `ipfs://` via dweb.link.
- **Wallets behind @thepark:** thepark.eth (`0x589f...0503`) + two linked creator wallets.
