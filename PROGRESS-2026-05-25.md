## What shipped, grouped by surface

### 🎵 Music player — two real bugs fixed

**Bug A — self-pause (commit `f8018e8`)**
The multi-tab `BroadcastChannel` coordination from the prior session had two separate channel instances in `GlobalPlayer.tsx` — one for sending, one for listening. Per spec, a `BroadcastChannel` only excludes its own *sender* instance from receiving its own messages; the *other* instance in the same tab still receives them. So every press of play immediately echoed back to the listener and paused itself.

**Fix:** single shared `BroadcastChannel` in a `useRef`, plus a `tabIdRef` guard so the listener ignores any message whose `tabId` matches its own (defense in depth for StrictMode or future refactors).

**Bug B — play/pause icon out of sync (commit `ef51fe3`)**
GlobalPlayer was syncing one direction only: store → Howler. When Howler's real state diverged (autoplay block, background-tab auto-pause, end of queue, network stall), the store never caught up — so the play/pause button showed the wrong icon.

**Fix:** wired Howler's events back to the store on both the primary and fallback Howl instances:
- `onplay` → `resume()`
- `onpause` → `pause()`
- `onend` → `pause() + setProgress(0)` when queue is empty, else `handleNext()`
- fallback `onloaderror` → `pause()`

No infinite loops because the sync effect short-circuits when `howl.playing()` already matches `isPlaying`.

### 🗳️ DAO — live auction now self-syncs

Before: `latestTokenId: 604` was hardcoded — stale within a day.

After (commit `2d9b172`): every page mount runs `findLiveAuctionTokenId(baseline, lookahead=10)` which probes `tokenURI(baseline + 0..10)` in parallel via `Promise.all`. Returns the highest contiguous existing tokenId — stops at the first gap so it never wastes RPC past the live edge.

- Baseline shipped at **610** (current auction as of 2026-05-25)
- Will advance forward automatically as new auctions mint
- Mirrored in both `components/dao/LatestAuction.tsx` (home strip) and `src/app/dao/content.tsx` (full /dao page)
- The "Bid on Nouns Builder" link, the "#N" number, and the rendered artwork all read from the resolved tokenId

The home page also surfaces the **Live Auction strip** below the CoverFlow so visitors see the current token without clicking into `/dao`.
